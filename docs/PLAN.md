## 2. 핵심 기능 정의 (Core Features)

### 2.1. 스터디 환경 설정 및 관리

| 기능 ID     | 기능명           | 상세 설명                                                        |
| :---------- | :--------------- | :--------------------------------------------------------------- |
| **STM-001** | 스터디 생성/참여 | 그룹장(운영자)이 스터디를 생성하고, 초대 코드를 통해 멤버가 참여 |
| **STM-002** | 15일 일정 등록   | Day 1 ~ Day 15의 학습 목표, 책 챕터, 필수 과제 내용을 미리 등록  |
| **STM-003** | 멤버 관리        | 스터디 멤버 리스트 확인, 진도율 및 제출 현황 모니터링            |

### 2.2. 개인 학습 및 기록 (사용자 기능)

| 기능 ID     | 기능명               | 상세 설명                                                                  |
| :---------- | :------------------- | :------------------------------------------------------------------------- |
| **USR-001** | **Day별 과제 제출**  | 해당 Day의 학습을 완료한 후, 필수 과제 답변을 텍스트 또는 파일로 제출      |
| **USR-002** | **느낀점/소감 기록** | 학습 내용에 대한 개인적인 요약 및 느낀점/소감을 자유 형식으로 작성         |
| **USR-003** | 공개 범위 설정       | 느낀점/소감 및 과제 제출 내용을 '멤버에게 공개' 또는 '나만 보기' 설정 가능 |
| **USR-004** | 나의 진도율 확인     | 전체 15일 중 완료한 Day의 비율(%)을 확인하는 대시보드                      |

### 2.3. 커뮤니티 및 피드백

| 기능 ID     | 기능명               | 상세 설명                                                                      |
| :---------- | :------------------- | :----------------------------------------------------------------------------- |
| **COM-001** | 멤버별 현황 대시보드 | 스터디 메인 페이지에서 모든 멤버의 Day별 진도율과 과제 제출 여부를 한눈에 확인 |
| **COM-002** | 피드백/댓글 기능     | 제출된 과제 및 느낀점 기록에 대해 다른 멤버나 운영자가 댓글로 피드백 제공      |

## 3. 화면 구성 및 흐름 (Wireframe/Flow)

### 3.1. 주요 화면

- **메인 대시보드:** 스터디 전체 진행률, 오늘의 과제, 멤버별 현황 요약
- **Day 상세 페이지 (Day 1~15):** 학습 목표, 과제 제출 폼, 느낀점/소감 작성 폼, 멤버별 제출물 열람 섹션
- **마이 페이지:** 개인 진도 캘린더, 내가 작성한 기록 모아보기
- **로그인/회원가입:** 스터디 참여를 위한 기본 인증 절차

### 3.2. 핵심 사용자 흐름 (Day별 과제 제출)

1.  **사용자 로그인** $\rightarrow$ (메인 대시보드에서) **오늘의 Day 확인**
2.  **Day 상세 페이지 이동** $\rightarrow$ 학습 목표/과제 내용 확인
3.  **과제 제출 폼 작성** (USR-001) $\rightarrow$ **느낀점/소감 폼 작성** (USR-002)
4.  **제출 버튼 클릭** $\rightarrow$ 나의 진도율 업데이트 (USR-004)
5.  **멤버별 현황 대시보드**에 나의 제출 여부 '완료'로 표시 (COM-001)
